## [首个无细胞骨架合成细胞成功生长分裂](https://www.quantamagazine.org/for-the-first-time-a-cell-built-from-scratch-grows-and-divides-20260701/) ⭐️ 9.0/10

科学家设计出名为 SpudCell 的合成细胞，无需细胞骨架即可生长分裂，首次在人工系统中实现细胞分裂。 这绕过了合成生物学中一大长期障碍——合成细胞的分裂难题，为创建可控的人工生命形式铺平道路，有望应用于医药、生物制造等领域。 SpudCell 利用脂质囊泡和最小遗传机制，通过物理力而非蛋白质细胞骨架实现分裂。该研究尚未正式发表，且在被期刊拒绝后引发了公关争议。

hackernews · defrost · 7月1日 14:20 · [社区讨论](https://news.ycombinator.com/item?id=48747304)

**背景**: 在天然细胞中，细胞骨架是由蛋白质纤维构成的网络，提供结构支撑并通过收缩驱动分裂。合成生物学致力于从零开始构建最小细胞，但此前无法实现分裂。SpudCell 通过使用脂质囊泡和替代机制绕过了这一限制，代表了一种简化的生命模型。

## [FFmpeg 9.1 发布新版 AAC 编码器，音质显著提高](https://hydrogenaudio.org/index.php/topic,129691.0.html) ⭐️ 8.0/10

FFmpeg 9.1 版本引入了一个全新的原生 AAC 编码器，大幅提升音质，解决了早期 FFmpeg AAC 编码器中长期存在的唧唧声等杂音问题。 这一更新使 FFmpeg 内置的 AAC 编码质量能够与 Apple Core Audio 等专有编码器相媲美，有利于视频录制、直播及任何需要高质量 AAC 输出而无须额外依赖的场景。 该编码器主要针对 48kHz 音频进行了优化，并绕过了立体声 PNS（感知噪声替换）中长期存在的解码器错误。调优过程很大程度上依赖于开发者主观的试听测试，凸显了音频编码优化的感知特性。

**背景**: AAC（高级音频编码）是一种有损音频压缩标准，常用于 MP4 容器和流媒体服务。FFmpeg 是一个广泛使用的开源多媒体框架。音频压缩伪影是编码过程中数据丢失导致的听觉失真。编解码器质量通常通过客观指标和主观听音测试来评估，基准测试在不同比特率和编码器之间比较性能。

## [Cloudflare 通过 HTTP 402 推出支付网关，实现微交易](https://blog.cloudflare.com/monetization-gateway/) ⭐️ 8.0/10

Cloudflare 发布了支付网关，利用长期未使用的 HTTP 402 状态码（x402）对其网络后的任何资源实现按请求收费，并以稳定币即时结算。 这可能为 API 和 AI 代理解锁微交易，允许无需预存账户的自动支付，并可能最终使 HTTP 402 标准在规模上得到实际采用。 该网关在 Cloudflare 遍布 330 多个城市的边缘网络上运行，以实现低延迟，目前支持稳定币支付。但按请求支付涉及的发票和税务等法律复杂性仍有待解决。

hackernews · Cloudflare Blog · 7月1日 13:59 · [社区讨论](https://news.ycombinator.com/item?id=48746914)

**背景**: HTTP 402“需要付费”是一个长期保留但很少使用的状态码，旨在用于数字支付。之前的微支付尝试因缺乏基础设施和用户采用而失败。Cloudflare 庞大的网络和 AI 代理的兴起为自动化按请求付费提供了新的场景，可能使其变得可行。

## [PEARL 系统零样本 OpenBind 夺冠，共折叠技术引领药物发现新范式](https://www.latent.space/p/the-coolest-diffusion-research-isnt) ⭐️ 8.0/10

Genesis Molecular AI 的 PEARL 系统在蛋白质-配体结合预测中实现了零样本条件下的 78%成功率，超越了所有共折叠模型，无需靶点特异性调优。这一突破与包括前 Meta AI 负责人 Sergey Edunov 在内的专家讨论，展示了扩散模型如何推动分子生物学发展。 PEARL 的零样本成功表明，扩散模型无需昂贵的实验数据即可可靠预测蛋白质-配体相互作用，极大地加速药物发现流程。这可能带来新疗法的更快开发，并对计算生物学产生广泛影响。 PEARL 仅使用蛋白质序列、配体 SMILES 和 apo 晶体结构作为输入，无需结合位点信息或靶点特异性训练。其 78%的成功率基于严格联合标准：RMSD≤2 Å、PoseBusters 有效性及 LDDT-PLI≥0.8，采用最佳 25 选策略。

rss · Latent Space · 7月1日 14:42

**背景**: 扩散模型最初在图像生成领域取得成功，现已越来越多地应用于分子生物学，用于分子对接和药物设计等任务。共折叠模型（如 AlphaFold3 和 RoseTTAFold All-Atom）预测蛋白质-配体复合物的联合 3D 结构，但通常需要靶点特异性训练。PEARL 的零样本方法消除了这一需求，有望使基于结构的药物发现更快、更易普及。OpenBind 基准测试使用 RMSD 和 LDDT-PLI 等严格指标评估蛋白质-配体对接精度。

## [MOTHRAG：无图多跳检索框架超越基于图的系统](https://www.reddit.com/r/MachineLearning/comments/1ukotww/p_mothretrieval_graphfree_multihop_retrieval_via/) ⭐️ 8.0/10

开源的 MOTHRAG 框架提出了一种无图多跳检索方法，利用稠密索引和查询时编排，在 HotpotQA、2WikiMultiHopQA 和 MuSiQue 基准上超越了 GraphRAG、HippoRAG 和 RAPTOR，同时支持增量数据更新，无需昂贵的重新索引。 通过消除离线知识图谱构建和重新索引的需求，MOTHRAG 大幅降低了频繁变化数据集上多跳 RAG 的成本和复杂度，使得在无 GPU 的商用 API 上实现精准多跳检索成为可行。 MOTHRAG 在 HotpotQA 上达到 78.1%的准确率，在 2WikiMultiHopQA 上达到 76.3%，但在 MuSiQue 上仅 50.5%，受限于检索召回瓶颈。每次查询成本约 0.03 美元，基于商用 API，并以 Apache 2.0 许可发布。

reddit · r/MachineLearning · /u/Annual-Commercial563 · 7月1日 15:26

**背景**: 多跳检索需要连接多个文档来回答复杂查询。基于图的 RAG 系统（如 GraphRAG 和 HippoRAG）离线构建知识图谱以捕获实体关系，但更新它们需要昂贵的 LLM 调用重新索引。MOTHRAG 在查询时使用稠密索引进行关系解析，避免了静态图结构。
